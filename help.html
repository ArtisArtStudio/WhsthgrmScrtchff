WhoHasIt — Help & Instructions

Overview

This is a personalized scratch-off party game meant for sharing with guests. You (the seller) will place a cropped head/face photo of the bride or groom into the game and give the buyer a link. Up to 20 players may join the game at once.

How it works — Seller / Setup (what I do for you)

  • Receive photo: The buyer sends you a clear photo of the person (head/face).
  • Crop the photo: You crop the image to show only the head/face at the correct aspect ratio for the game.
  • Place photo into the game: You upload or embed the cropped face into the game template and generate a game link for the buyer.
  • Share link: Send the buyer the game link and any instructions for hosting (see Host steps below).

How You & Players Use It

  1. Open the game link in a browser on a device (phone/tablet/desktop).
  2. The person who will be the host should open the link first and start the game session (this may enable host-only controls such as resetting). Preferably this could be the person who bought this product.
  3. Other players join by opening the same link. There is a maximum of 20 players at once.
  4. Each player can scratch the circle to reveal and guess the person. The UI shows the scratch canvas and the control area (such as Reset buttons and a host-only checkbox).

Resetting the game

There are two reset controls available in the UI:

  • Scratch Again — resets only the scratch area for the current player so they can re-scratch.
  • Reset Game (Host only) — clears game state for all players and returns the session to the start. Only the host can perform a full reset; use this when the player list should be cleared or you need to start a new round.

Host-only option: there is a checkbox ("Clear host on reset") that, when checked, will also clear host assignment when the Reset Game button is used. If left unchecked, the same host remains after reset. Only the host will see and be able to use the full Reset Game control.

Troubleshooting & Common Errors

Game reports "Full" or "Game is full":

Cause: The game identifies players by the browser/tab session. There are two common situations that trigger a "game full" message.

  1. Previously registered player reconnects using a different browser, device, or tab: The server still counts the original session as connected, so the new connection may be blocked with a "game full" message. Fix: the host should perform a Reset Game to clear all sessions and allow everyone to rejoin. Warning: performing a full reset will clear the current session state and may change who the winner is.
  2. An additional user attempts to join but the current connected player count equals the configured max players: If the lobby is at its current limit, new players cannot join. Fix: if the host increases the player limit (and it is below the absolute cap of 20), the additional player can join. Note: if the game is already at the absolute cap (20), new players cannot be added.

Additional fix: if a returning player reopens the same browser tab they used originally (or avoids switching browsers), they may reconnect without requiring a host reset.

Other error cases & fixes

  • Image not centered or cropped oddly: Re-send a properly cropped head/face photo. Crop so the face is centered with a little margin around the head.
  • Scratch area not responding: Ensure JavaScript is enabled and that the page is not blocked by an ad or privacy blocker. Try reloading the page (same tab) and verify touch/mouse interaction permission.
  • Audio or confetti not playing: Some browsers block autoplay of audio. The host/player may need to interact with the page (tap/click) to enable sound, or enable sound in the initial modal prompt if present.
  • Buttons hidden or under the scratch circle: If layout issues occur after rotating the device, refresh the page or open the link in the original orientation. If problems persist, the host can reset the game which will reposition UI elements for new sessions.
  • Players cannot join: Confirm the host started the session and that the link is correct. Make sure fewer than 20 players are currently connected (or ask the host to Reset Game).

Shown Error Messages & Fixes

Below are specific error messages the game may display and suggested fixes.

  • "Failed to fetch" / Network / CORS errors: The browser could not contact the game server or the response was blocked. Fix: check network, reload the page (same tab), ensure the GAS endpoint is reachable, and try again. The host can perform a full Reset Game if issues persist.
  • "No game ID provided in URL": The page was opened without the required ?gameID=... parameter. Fix: use the full link provided by the seller including the gameID query string.
  • "Game not found: ...": The server does not recognize the provided game ID. Fix: verify the link, ask the seller to confirm the game was created, or request a new link.
  • "Failed to register: ...": The server refused or could not complete player registration. Fix: retry in the same browser/tab; if the error indicates the game is full, ask the host to reset or increase the player limit.
  • "Failed to load image: ..." or "Unable to load groom photo; using default image": The supplied groom/bride image could not be loaded. Fix: check your internet connection; contact support if not.
  • "Failed to load default images: ...": The app couldn't load fallback images from the site. Fix: check your internet connection; contact support if not.
  • "This browser does not support canvas": The user's browser is too old or disabled required features. Fix: open the game in a modern browser (Chrome, Firefox, Edge, Safari) and enable JavaScript.
  • "Timeout waiting for host to start the game": The lobby waited too long for the host to start. Fix: confirm the host opened the link and pressed Start; refresh and rejoin if needed.
  • "You were removed from the lobby": The server removed that player (kicked). Fix: rejoin using the same browser/tab or ask the host why you were removed.
  • Host messages like "Failed to start game", "Failed to update", or "Reset error: ...": These indicate server-side or network problems during host actions. Fix: retry the action, check network connectivity, and if persistent, perform a full Reset Game or contact support.
  • "Not all players have finished scratching yet. Please wait for everyone to reveal their result.": Host attempted reset before everyone finished. Fix: wait until all players finish, or coordinate participants to finish before resetting.
  • "Game is full" Please refer to the previous red highlighted section.
  • "Lobby is full": This can appear when the host reduces the configured player limit and the server removes one or more previously registered players to meet the new limit. Affected players may see the lobby/full message because their session was invalidated by the server. Fix: the host should either increase the player limit back or perform a full Reset Game so everyone can rejoin. Warning: a full reset clears session state and may change who the winner is. If an extra (new) player wants to join but the lobby is at the configured limit, the host may increase the limit (if below 20) to admit them.

Support & Best Practices

  • Tell buyers to open the game link on the device they plan to use for joining and to avoid switching browsers/tabs mid-game.
  • Keep a copy of the cropped face image in case you need to re-embed it for a new session.
  • If the buyer reports persistent issues, ask for a screenshot and the browser name/version so you can reproduce locally.
